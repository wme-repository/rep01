# Plan 07-02: Settings Form for Environment Variables

**Wave:** 1
**Depends on:** 07-01
**Files modified:** `src/server/index.js`, `frontend/src/api.js`, `frontend/src/components/Settings.jsx`, `frontend/src/App.jsx`

<objective>
Add API endpoints to read/write .env file and create a Settings UI tab to edit environment variables (competitor URLs, API keys, etc.).
</objective>

<read_first>
- `src/server/index.js` — Express server
- `.env.example` — all available environment variables
- `frontend/src/api.js` — API client
</read_first>

<action>
1. **Add server endpoints** to `src/server/index.js`:
   - `GET /api/settings` — reads and returns .env as JSON
   - `PUT /api/settings` — receives JSON and writes to .env file

2. **Update API client** in `frontend/src/api.js`:
   ```js
   getSettings: () => fetch(`${API_BASE}/settings`).then(r => r.json()),
   updateSettings: (data) => fetch(`${API_BASE}/settings`, {
     method: 'PUT',
     headers: { 'Content-Type': 'application/json' },
     body: JSON.stringify(data)
   }).then(r => r.json()),
   ```

3. **Create Settings component** (`frontend/src/components/Settings.jsx`):
   - Tabbed form: "Geral", "Competidores", "APIs"
   - Fields grouped by category:
     - **Geral:** LOG_LEVEL, DEBUG_MODE
     - **Competidores:** COMPETITOR_URL_1, COMPETITOR_URL_2, COMPETITOR_URL_3, MAIN_SITE_URL
     - **APIs:** ARVO_API_KEY, WORDPRESS_URL, WORDPRESS_USERNAME, WORDPRESS_PASSWORD, BLOTADO_API_KEY
   - Auto-save on blur with success/error toast
   - Load initial values from GET /api/settings

4. **Add Settings tab** to `App.jsx` navigation and render `<Settings />`

5. **Add input validation:**
   - URLs must start with http:// or https://
   - Required fields show error if empty on save
</action>

<acceptance_criteria>
- [ ] GET /api/settings returns all .env variables as JSON
- [ ] PUT /api/settings writes JSON back to .env file
- [ ] Settings tab shows form with all variable categories
- [ ] Editing a field and blurring saves successfully
- [ ] Error toast shown if save fails (e.g., file permission)
- [ ] Settings tab accessible from main navigation
</acceptance_criteria>
