## Tasks

### Task 1: Create .env.example

**read_first:**
- .planning/REQUIREMENTS.md (all env vars mentioned in requirements)
- .planning/research/STACK.md (env handling approach)

**acceptance_criteria:**
- .env.example exists in project root
- File contains BUSINESS_NAME, TARGET_AUDIENCE, BUSINESS_GOAL, MAIN_SITE_URL
- File contains ARVO_API_KEY, WORDPRESS_URL, WORDPRESS_USER, WORDPRESS_APP_PASSWORD
- File contains BLOTADO_API_KEY, BLOTADO_BLOG_ID, RSS_FEED_URL
- File contains APPROVAL_MODE (auto ou manual)
- File contains COMPETITORS (comma-separated URLs)
- No actual API keys or passwords in the file
- Comments explain each variable

**action:**
Create .env.example:
```
# ===========================================
# SEO Ranker - Environment Configuration
# ===========================================
# Copy this file to .env and fill in your values.
# NEVER commit .env to version control!

# ---------- Business Context ----------
BUSINESS_NAME="Your Business Name"
TARGET_AUDIENCE="Your Target Audience Description"
BUSINESS_GOAL="leads|sales|traffic"
MAIN_SITE_URL="https://yoursite.com"
CMS_TYPE="wordpress"

# ---------- Competitors ----------
# Comma-separated URLs (no spaces)
COMPETITORS="https://competitor1.com,https://competitor2.com"

# ---------- Arvo API (Content Generation) ----------
# Get your API key from https://arvo.com/api
ARVO_API_KEY="your_arvo_api_key_here"

# ---------- WordPress ----------
# Your WordPress site URL (no trailing slash)
WORDPRESS_URL="https://yoursite.com"
# WordPress username (for Application Password auth)
WORDPRESS_USER="your_username"
# WordPress Application Password (NOT your regular password)
# Generate at: WordPress Admin > Users > Application Passwords
WORDPRESS_APP_PASSWORD="xxxx xxxx xxxx xxxx xxxx xxxx"

# ---------- Blotado (Social Media) ----------
# Get from your Blotado dashboard
BLOTADO_API_KEY="your_blotado_api_key"
BLOTADO_BLOG_ID="your_blog_id"

# ---------- RSS Feed ----------
# Full URL to your blog's RSS feed
RSS_FEED_URL="https://yoursite.com/feed"

# ---------- Operational ----------
# "auto" = publish without confirmation
# "manual" = require approval before each publish
APPROVAL_MODE="manual"

# ---------- Paths ----------
# Absolute path to context directory (for persistence)
CONTEXT_PATH="./context"
# Absolute path to logs directory
LOGS_PATH="./logs"
# Absolute path to brand kit directory
BRAND_KIT_PATH="./brand-kit"

# ---------- Execution Mode ----------
# "true" = verbose output for debugging
DEBUG="false"
```
